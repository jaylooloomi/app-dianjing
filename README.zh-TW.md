> 點睛(出自「畫龍點睛」—— 那關鍵的一筆讓作品活起來)是一款跨平台桌面小工具。在**任何** app 裡選取文字、按一個快捷鍵,**本地的小型語言模型**就會把它改寫成符合當下場合的版本 —— 輕鬆的 LINE 訊息、正式的 Email、給主管的回報、吸睛的社群貼文,或是優化過的 AI 提示詞。它**完全在你的機器上運作**(免裝 Ollama、無常駐服務、不需 API 金鑰),你的文字不會被送到任何地方。模型在首次啟動時自動下載,之後就**100% 離線**。

---

## 痛點

現有的「AI 改寫」功能都被圈在牆內:

- **只活在單一 app 裡。** 某個編輯器或網頁信箱有「潤飾」鈕,但你在聊天、表單、工單、筆記裡打的字只能自己來。
- **雲端改寫等於把文字交出去。** 潤飾一份工作回報或私人訊息,通常得貼到別人的伺服器上。
- **「變好一點」忽略情境。** 給朋友的訊息和給主管的回報語氣不同 —— 一個按鈕套不了所有場合。
- **不斷切換視窗。** 離開手邊工作 → 開聊天機器人 → 貼上 → 把結果複製回來 → 自己取代。

## 解法

**點睛把這些全部收進一個快捷鍵,而且在你自己的機器上。**

```
選取文字  →  右 Alt  →  選情境  →  取代
                  ↓
      本地 AI 幾秒內就地改寫,完全離線。
```

不用終端機、不用 API 金鑰、不連雲端。模型首次啟動自動安裝,每次改寫都在**你的 CPU 上、程式內**完成 —— 然後把結果直接放回你原本打字的地方。

## 主要功能

- 🎯 **在任何 app 都能用。** 全域 `右 Alt` 抓取你選取的文字 —— 聊天、瀏覽器、編輯器、表單,免複製貼上。
- 🏠 **100% 本地、隱私。** 用 `node-llama-cpp` 在程式內跑 `Qwen2.5-1.5B-Instruct`。免 Ollama、無常駐服務、不需 API 金鑰。模型下載一次後即可離線。**你的文字不離開機器。**
- 🎬 **依情境調整語氣。** 內建六種情境 —— 一般、LINE、Email、社群貼文、正式回報、提示詞優化,各有各的口吻。
- 🌐 **回覆語言可選。** 潤飾輸出可選繁體中文 / 简体中文 / English,與介面語言獨立。
- ✍️ **修錯字 + 調語氣,但不亂編。** 提示詞經過調校,專注修正錯誤、貼合情境,不會自行虛構收件人、人名或事實。
- 🔁 **就地取代或複製。** 結果可編輯;按「取代原文」貼回選取處,或「複製」帶走。
- 🪶 **輕量、不擋路。** 無邊框、置頂、**自動貼齊內容高度**的小視窗;常駐背景,關閉即隱藏。
- 🔒 **零遙測。** 不收集任何使用資料。

## 運作方式

```
在任何 app 選取文字
        │  右 Alt
        ▼
┌─────────────────────────────────┐   選情境 + 回覆語言
│  點睛視窗 (Electron)            │ ──────────────┐
└─────────────────────────────────┘               │
        ▲ 串流回傳結果                              │ 擷取選取文字
        │                                          ▼ (借用剪貼簿,事後還原)
┌──────────────────────────────────────────────────────────┐
│  本地 LLM,程式內運作 (node-llama-cpp)                     │
│  Qwen2.5-1.5B-Instruct · CPU · 無常駐服務 · 完全離線        │
└──────────────────────────────────────────────────────────┘
        │
        ▼
   就地取代   /   複製
```

全域 `右 Alt` 由 `uiohook-napi` 攔截。點睛會短暫複製你的選取(隨即還原你原本的剪貼簿),只把那段文字送進**載入在程式內**的 GGUF 模型,把潤飾結果串流到一個可編輯的框裡,你確認後再貼回原處覆蓋選取。首次啟動會自動從 Hugging Face 取得模型(約 1.1 GB),之後全程離線。

## 情境

| 情境 | 用途 |
|---|---|
| **一般** | 修正錯別字、標點與語法;保持中性自然、貼近原文。 |
| **LINE** | 口語、親切的即時訊息語氣;適度使用表情符號。 |
| **Email** | 專業的 email 語氣,搭配不具名的通用問候與結尾。 |
| **社群貼文** | 精煉、有節奏、好讀;適合處加上 hashtag。 |
| **正式回報** | 對主管回報:重點先行、客觀精簡,不加生硬稱謂。 |
| **提示詞優化** | 把粗略/含糊的指令改寫成更清楚、更有效的 AI 提示詞(重構模式,而非最小潤飾)。 |

回覆語言(繁中 / 簡中 / English)在視窗標題列另外選擇。

## 安裝

> 目前還沒有簽章過的安裝檔 —— 請從原始碼執行,或自行建置本機 app。

**從原始碼執行**

```bash
git clone https://github.com/jaylooloomi/app-dianjing.git
cd app-dianjing
npm install
npm start            # 首次啟動會下載模型(約 1.1 GB)並預熱
```

**建置桌面 app**

```bash
npm run dist:win     # Windows(dist/ 內的免安裝 app)
npm run dist:mac     # macOS(.dmg)
```

`node-llama-cpp` 已提供 Windows 與 macOS 的預編譯二進位檔,一般安裝不需要原生編譯工具鏈。

## 使用方式

1. 啟動點睛 —— 它常駐在背景(不主動開視窗,等你叫它)。
2. 在任何 app 裡**選取**要潤飾的文字。
3. 按 **`右 Alt`**,視窗出現,你的文字會成為「原文」。
4. 選一個**情境**與**回覆語言**(預設繁體中文)。
5. 檢視/編輯結果,然後按**取代原文**或**複製**。

| 快捷鍵 | 動作 |
|---|---|
| `右 Alt`(全域) | 擷取目前選取 → 開啟視窗 |
| 點情境按鈕 | 切換情境並重新潤飾 |
| `Esc` | 關閉視窗 |

> 什麼都沒選就按 `右 Alt`,它只會提醒你先選取文字。

## 隱私與安全

- **完全本地推論。** 文字只送到程式內運作的模型 —— 不送任何伺服器。無帳號、無 API 金鑰。
- **首次之後離線。** 唯一的網路存取就是那一次模型下載。
- **剪貼簿只是借用,不會清掉。** 擷取選取時會先存下你現有的剪貼簿,事後還原。
- **零遙測。** 不收集也不傳送任何使用資料。

---

## 工程亮點

- **程式內 GGUF 推論** —— 透過 `node-llama-cpp`(N-API,在 Electron 下 ABI 穩定);免 Ollama、無 sidecar 常駐、可純 CPU 跑;模型首次啟動自動從 Hugging Face 下載。
- **單獨 `右 Alt` 全域快捷鍵** —— Electron 的 `globalShortcut` 無法綁定單獨的右 Alt,改用 `uiohook-napi`;在 key-*up* 觸發並偵測「乾淨單按」,確保不會在 `Alt` 組合鍵時誤觸。
- **跨平台擷取與取代** —— Windows 用常駐 PowerShell(前景視窗存/還原 + 模擬 `Ctrl+C`/`Ctrl+V`)、macOS 用 `osascript`、Linux 用 `xdotool`;過程保留你原本的剪貼簿。
- **自動貼齊內容的視窗** —— 以 `ResizeObserver` 量測內容,視窗用 `setContentSize` 跟著調整,永遠不留白;無邊框、置頂、關閉即隱藏而非結束。
- **純函式的提示詞建構器**,並有單元測試涵蓋回覆語言與 潤飾/重構 兩種模式。

## 已知限制

- **它是 1.5B 模型。** 兩個已知的粗糙處:**提示詞優化**情境有時會去「回答」指令而不是「改寫」它;以及某些情境下**英文輸出偶爾會跑回中文**。較大 / 可選的模型(見藍圖)是根本解。
- **首次啟動下載約 1.1 GB。** 需要一次網路;之後離線。
- **跨 app 擷取仰賴剪貼簿** + 模擬的複製鍵 —— 少數複製處理非標準的 app 可能擷取不到。
- **macOS** 需要「輔助使用」權限(全域快捷鍵 + 模擬鍵盤);**Linux** 擷取使用 `xdotool`(X11)。

## 開發

```
app-dianjing/
├── main.js                 ← Electron 主程序:快捷鍵、視窗、IPC、模型生命週期
├── preload.js              ← 提供給視窗的 contextBridge API

    ├── popup.html / popup.js   ← 潤飾介面(情境、語言、結果)
    ├── polishPrompts.js        ← 情境定義 + 提示詞建構(純函式)
    ├── polishService.js        ← node-llama-cpp 載入 + 串流潤飾
    ├── modelManager.js         ← 首次啟動下載模型(resolveModelFile)
    └── textCapture.js          ← 跨平台「擷取選取 / 貼回」
```

```bash
npm start      # 開發執行
npm test       # 單元測試(node --test)
```

## 系統需求

- **Windows 10/11** 或 **macOS**(Apple Silicon 或 Intel);擷取功能在 **Linux**(X11)。
- **約 1.5 GB 可用磁碟**(模型約 1.1 GB)。
- 首次下載模型需要**網路**;之後離線即可。
- 一般 CPU 即可 —— 不需 GPU。
